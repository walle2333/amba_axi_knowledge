# AMBA AXI Knowledge Base

这是一个面向长期维护的 AMBA AXI 本地知识库，目标是系统整理 AXI 基础知识、进阶协议细节、SoC 芯片实践、RTL 示例、验证方法和可靠原始参考资料。

## 维护目标

- 中文为主，关键协议术语保留英文。
- 原始参考资料保存到 `sources/`，结构化笔记保存到 `notes/`。
- 重要结论优先引用 Arm 官方规格或厂商公开文档。
- 示例覆盖协议行为、RTL 模式和 SoC 架构场景。
- 使用 Git 维护演进历史。

## 目录说明

```text
sources/    原始参考资料，按来源分类保存，不直接修改。
notes/      结构化知识笔记，中文为主。
examples/   时序、事务、RTL、SoC 场景和验证示例。
summaries/  速查表、检查表和调试总结。
metadata/   资料索引、术语表、阅读计划和待办事项。
scripts/    后续可能加入的资料处理脚本说明。
```

## 资料可靠性分级

```text
primary    Arm 官方规格书、Arm 官方技术文档、官方培训资料。
secondary  EDA/IP/FPGA/SoC 厂商公开文档、白皮书、应用说明。
tertiary   博客、课程笔记、GitHub 项目文档、论坛讨论。
```

协议规则以 `primary` 为最高优先级。工程经验可以参考 `secondary`。`tertiary` 只作为辅助理解，不能作为关键协议结论的唯一依据。

## 新增资料流程

1. 将原始文件保存到 `sources/` 对应分类目录。
2. 在 `metadata/sources.yaml` 登记来源、URL、访问日期、版本、可靠性和本地路径。
3. 如果资料无法直接下载，登记 URL、访问日期和限制原因。
4. 在相关笔记底部列出引用资料 ID。

## 笔记写作格式

每篇主题笔记建议包含：

```text
目的
核心概念
相关信号
协议规则
具体示例
SoC 实践注意点
常见问题
参考资料
```
